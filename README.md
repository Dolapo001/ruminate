# Ruminate — Explainable AI for Dairy Herds (full stack)

Sensor-based prediction of **health and reproductive traits** in Nigerian dairy
cows, with an **explanation behind every alert**. Built for White Fulani and
Holstein-Friesian crossbreds, and designed to tell heat stress apart from disease.

Three parts, one system:

Three parts, one system:

```text
ai/        synthetic data → features → train (XGBoost vs black-box vs glass-box) → SHAP
backend/   Django + DRF: Multi-tenant API, models, and real-time sensor simulation
frontend/  Next.js (Savanna Futurism UI): Role-based dashboards for Vets and Farmers
```

The data flows: sensors → features → **XGBoost** predicts {Healthy, At Risk, In
Estrus} → **SHAP** explains each prediction → Django serves it → the dashboard
shows the *why* → the vet confirms → that becomes new ground truth.

---

## What the AI actually achieves (real numbers, grouped split, label-noise)

| Model | Accuracy | Macro-F1 | Macro-AUC |
|-------|:--------:|:--------:|:---------:|
| Decision Tree (glass-box) | 0.86 | 0.66 | 0.78 |
| Neural Net (black box) | 0.95 | 0.75 | 0.71 |
| **XGBoost + SHAP (ours)** | **0.95** | **0.80** | **0.81** |

The interpretable model **matches the black box on accuracy and beats it on
F1/AUC** — while remaining fully explainable. See `ai/outputs/` for the
confusion matrix, model comparison, and SHAP summary figures.

## Run it

### 1 — AI pipeline (optional; trained artifacts are already included)
```bash
cd ai
pip install -r requirements.txt
python generate_data.py        # synthetic 15-min sensor data
python features.py             # daily engineered features
python train.py                # trains + evaluates + SHAP + saves model_xgb.joblib
python make_snapshot.py         # one realistic "this morning" snapshot
python lstm_cv.py               # LSTM + 5-fold grouped cross-validation (needs tensorflow-cpu)
```

### 2 — Backend (Django)
```bash
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_herd      # scores the snapshot with the model → Animals + Alerts
python manage.py runserver      # http://localhost:8000
```

### 3 — Frontend (Next.js)
```bash
cd frontend
npm install
cp .env.local.example .env.local      # points at http://localhost:8000
npm run dev                            # http://localhost:3000
```

Open **http://localhost:3000**. The dashboard now shows **real model
predictions** with **real SHAP reason-bars**. Leave `NEXT_PUBLIC_API_URL` unset
to run the frontend standalone on built-in mock data.

## The thesis, shown live

Open cow **Amina (NG-014)**. The model flags *Possible mastitis (97%)* and the
explanation reads: milk conductivity ↑ (the decider — heat never raises it),
rumination ↓, plus a **teal "Weather (THI) — rules out heat"** bar. That teal bar
is the model using mild weather to confirm it's genuine disease, not tropical
heat stress — generated by SHAP on the real model, not hard-coded.

## API (consumed by the frontend)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/health` | service + model health check |
| GET | `/api/herd/summary` | herd-health counts |
| GET | `/api/cows` | herd list (supports `?farm_id=` filtering) |
| GET | `/api/cows/{id}` | one cow: vitals, SHAP reasons, timeline |
| GET | `/api/cows/{id}/history` | daily sensor history for the trend charts |
| GET | `/api/alerts` | alert feed (urgency-sorted) |
| GET | `/api/alerts/{id}` | full alert |
| POST | `/api/cows/{id}/diagnosis` | vet ground truth → resolves the alert |
| POST | `/api/farms` | registers a new farm (multi-tenant) |
| POST | `/api/cows/new` | registers a new cow to a farm |
| POST | `/api/cows/{id}/simulate` | triggers ML synthetic data generation for new cows |
| POST | `/api/ingest?score=1` | gateway store-and-forward: idempotent readings + on-arrival scoring |
| GET | `/api/metrics` | model comparison, importance, confusion matrix |

## What's included (production polish)

- **Role-Based Workflows** — A global `/dashboard` for Veterinarians to monitor all connected farms, and a strictly scoped `/farm` portal for individual Farmers to manage only their livestock.
- **Onboarding & Simulation** — Farmers can register new cows and click "Simulate Sensor" to instantly trigger the AI pipeline, generating 15 days of synthetic physiological data on-the-fly and activating the AI dashboard.
- **Per-cow trend charts** — real daily sensor history (rumination, body temp, activity, conductivity) with estrus/health event markers, animated.
- **Escalation Protocol** — Farmers have a "Call Vet" escalation button, while Vets have clinical diagnostic forms and deep Counterfactual Analysis tools to override or confirm AI findings.
- **100% Mobile Responsive** — The Savanna Futurism design system has been fully optimized down to a 320px width for Farmers out in the field.
- **Loading skeletons** on every data route, plus an "all clear" empty state.
- **Model & evaluation page** rendering live metrics, global SHAP importance, and the confusion matrix.

## Notes

- Data is **synthetic** (no public Nigerian sensor dataset exists) but encodes
  documented physiology, including heat stress as a deliberate confounder. Swap
  in a real CSV with the same columns and the pipeline runs unchanged.
- Label noise (~4.5%) simulates imperfect vet logs, which is why accuracy is a
  realistic ~95% rather than a suspicious 100%.


## v2 additions

- **LSTM + 5-fold cross-validation.** A true temporal model over the raw
  15-minute sequences, plus grouped CV so every score has a mean ± std. Result:
  interpretable **XGBoost still wins** (0.945 acc) over the black-box NN and the
  LSTM (0.913) — explainability costs nothing. Numbers show on the Model page.
- **Real store-and-forward ingest.** `POST /api/ingest` is idempotent on
  (animal, timestamp): re-sending a buffered batch after a network drop updates
  in place, never duplicates. With `?score=1` it aggregates the day and re-scores
  with the model. Try it:
  ```bash
  python backend/tools/gateway_sim.py          # sends a batch, then re-sends to prove dedup
  ```
- **PWA / offline.** Installable, a service worker serves the app shell and last
  data when the network is down, and vet diagnoses logged offline are **queued
  and synced on reconnect** — honouring the Chapter 3 resilience requirement.
